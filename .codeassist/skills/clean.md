# Code Cleanup Skill

Clean up code to improve maintainability and consistency.

## Steps

1. **Analyze the code** - Identify cleanup opportunities
2. **Remove dead code:**
   - Unused imports
   - Commented-out code
   - Unreachable branches
   - Unused variables/functions
3. **Organize structure:**
   - Sort imports
   - Group related functions
   - Remove duplicate code
4. **Apply style consistency:**
   - Consistent naming conventions
   - Consistent formatting
   - Consistent patterns

## Cleanup Checklist

- [ ] Remove unused imports
- [ ] Remove commented-out code
- [ ] Delete unreachable code paths
- [ ] Consolidate duplicate logic
- [ ] Sort imports (stdlib, third-party, local)
- [ ] Apply consistent naming
- [ ] Remove debug print statements

## Guidelines

- Don't change functionality
- Preserve all comments that explain WHY
- Keep TODO comments for tracking
- Make changes incrementally
